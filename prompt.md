# Daily-Box

**Role:** 你是一位顶尖的前端工程师和 UI/UX 设计师，擅长构建极简、优雅且功能强大的个人效率工具。

**Task:** 请为我编写一个单页面 Web 应用（SPA）的代码，要求可以直接部署在 GitHub Pages 上。这是一个名为 "Personal Nexus" 的日程与任务规划系统。

**1. 视觉风格 (Visual Identity):**

- **美学风格：** 极简主义、磨砂玻璃（Glassmorphism）感。使用深色模式（Dark Mode）为主，配以柔和的强调色（如：冰蓝色或极光紫）。
- **布局：** 响应式 Bento Grid（便当盒）布局，通过不同的卡片（Card）区分功能块。
- **动效：** 使用流畅的过渡动画（如进入页面的淡入效果、点击卡片的缩放反馈）。

**2. 核心功能模块 (Modules):**

- **Inbox (灵感/待办集散地):** * 支持快速输入任务。
  - 属性包含：优先级（P1-P4）、进度条（0-100%）、截止日期（DDL）。
- **Project Hub (项目跟踪):**
  - **草稿/脑暴区：** 一个白板风格的区域，支持简单的图形绘制或自由排版节点。
  - **里程碑路线图：** 垂直或水平的时间轴，标记关键节点。
- **Cold Storage (冷冻库):** * 专门存放长期计划或“以后再说”的想法。卡片视觉上应较暗，减少视觉干扰。
- **Time Planner (周/月计划):** * 一个简洁的日历视图或周列表，突出显示最重要的 3 件事。

**3. 交互与技术要求 (Technical Specs):**

- **直接交互：** 所有文本块、日期、进度条必须支持 **Click-to-Edit**（点击即编辑），修改后立即自动保存。
- **数据持久化：** 由于部署在 GitHub Pages（静态页面），请使用 **LocalStorage** 或 **IndexedDB** 实现本地数据持久化，确保刷新页面后数据不丢失。
- **技术栈：** 请使用 React 或 Vue 3 (CDN 版本)，结合 Tailwind CSS 进行样式开发。代码需要集成在一个 HTML 文件内或提供清晰的文件结构。
- **无代码修改：** 用户界面必须包含所有必要的 UI 组件来添加/删除/修改内容，不需要用户通过修改源代码来更新计划。
