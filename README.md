# DoreNexus

DoreNexus 是一个可直接部署在 GitHub Pages 的单页个人规划系统，采用窗口化交互，支持任务管理、并行项目计划、长期想法存储、时间规划与跨设备数据迁移。

## 功能概览

- Linux 风格窗口系统：
- 多窗口并存
- 可最小化到任务栏
- 可拖拽、可调整大小（桌面端）
- Inbox 收件箱：
- 快速创建任务
- 优先级（P1-P4）、进度、DDL
- 点击编辑（Click-to-Edit）
- DDL 自定义日历选择（支持切换月份）
- Project Hub 项目中心：
- 多计划并行切换
- 草稿白板节点（创建、删除、拖拽）
- 里程碑时间线（状态与日期）
- Cold Storage 冷存储：长期计划与想法低干扰保存
- Time Planner 时间规划：周/月计划 + Top3 重点
- 视觉系统：
- 深海极简 / 冷灰工业 两套主题一键切换
- 流体光效与鼠标视差跟随
- 数据能力：
- LocalStorage 自动保存
- Sync Vault 同步码导出/导入
- JSON 备份文件导出/导入

## 技术栈

- Vue 3（CDN）
- Tailwind CSS（CDN）
- 单文件应用：index.html

## 本地使用

1. 克隆仓库后直接打开 index.html 即可使用。
2. 建议使用本地静态服务预览（可避免部分浏览器本地文件策略限制）。

示例（任选其一）：

```bash
# Python
python -m http.server 5500
```

```bash
# Node (需先安装 serve)
npx serve .
```

然后访问：

- http://localhost:5500

## 部署到 GitHub Pages

1. 将代码推送到 GitHub 仓库。
2. 打开仓库 Settings -> Pages。
3. Source 选择 Deploy from a branch。
4. Branch 选择 main（或你的默认分支）+ /(root)。
5. 保存后等待构建完成，访问生成的 Pages 链接。

## 使用说明

### 窗口与任务栏

- 点击首页模块卡片打开对应窗口。
- 窗口标题栏可拖动（桌面端）。
- 右下角可缩放窗口（桌面端）。
- 左上黄点最小化窗口到任务栏。
- 任务栏点击窗口按钮可恢复/切换焦点。

### Inbox 的 DDL 日历

- 在创建任务区域点击“点击选择日期”。
- 在弹出的日历中切换上月/下月。
- 直接点击日期即写入 DDL。

### Project Hub 多计划

- 使用计划下拉框切换当前计划。
- 可新建、复制、删除计划。
- 当前计划的白板节点和里程碑互相独立保存。

### 主题切换

- 在底部任务栏点击主题按钮切换：
- 深海极简
- 冷灰工业

## 跨设备数据迁移

DoreNexus 是纯前端静态应用，不依赖后端数据库。跨设备可通过以下方式迁移数据：

1. 同步码（适合快速迁移）
- 在设备 A 打开 Sync Vault，生成同步码并复制。
- 在设备 B 打开 Sync Vault，粘贴同步码并应用。

2. 备份文件（适合长期归档）
- 在设备 A 导出 JSON 备份文件。
- 在设备 B 导入该备份文件。

## 数据存储说明

- 应用会自动将数据保存到浏览器 LocalStorage。
- 清除浏览器站点数据会导致本地数据被删除。
- 建议定期导出备份文件。

## 已知限制

- 移动端为可用性优先：窗口拖拽与缩放默认关闭。
- 同步码为手动迁移，不是实时云同步。
